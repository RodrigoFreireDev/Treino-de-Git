Para renomear um arquivo num repositório Git, utilize o comando git mv seguido do nome antigo e do novo nome do arquivo, por exemplo: git mv nome_antigo.txt novo_nome.txt. Este comando move o arquivo e adiciona a alteração ao "staging area", para que possa fazer o commit e salvar o histórico corretamente. 
Como Renomear um Arquivo com git mv 
Abra o terminal: no diretório do seu repositório Git.
Execute o comando: git mv, especificando o nome atual do arquivo e o nome que você deseja.
Exemplo: Se você quer renomear o arquivo documento.txt para relatorio.txt, o comando seria:
Código

        git mv documento.txt relatorio.txt
Verifique o estado: do repositório com git status. Você verá que o Git já preparou a mudança (a renomeação) para o "staging area". 
Exemplo de saída de git status:
Código

        Changes to be committed:
          (use "git reset HEAD <file>..." to unstage)

        	renamed:    documento.txt -> relatorio.txt
Faça o commit: da alteração para confirmar a renomeação no histórico do seu repositório. 
Comando de commit:
Código

        git commit -m "Renomeia documento.txt para relatorio.txt"
Envie as alterações: para o repositório remoto, se necessário. 
Comando de push:
Código

        git push origin <nome-da-sua-branch>
Porquê usar git mv?
Preserva o histórico: Usar git mv garante que o Git reconheça a mudança como uma renomeação, e não como a exclusão de um arquivo e a criação de outro. 
Simplifica o fluxo de trabalho: Combina as operações de remoção e adição do arquivo em um único comando, preparando a alteração automaticamente para o "staging area".
